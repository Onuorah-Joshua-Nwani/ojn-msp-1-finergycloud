Security Policy

1. Mobile App Security

The FinergyCloud mobile app is designed with security as a core principle. We implement multiple layers of protection to safeguard your data and ensure a secure user experience.

Secure Development

Our app is developed following secure coding practices and undergoes regular security testing before each release.

Secure Communications

All communication between the app and our servers is encrypted using TLS 1.2 or higher to protect data in transit.

Certificate Pinning

We implement certificate pinning to prevent man-in-the-middle attacks and ensure secure connections.

Vulnerability Management

We regularly scan for vulnerabilities and promptly address security issues through app updates.

2. Data Protection

2.1 Data Encryption

We protect your data using industry-standard encryption:

  • All sensitive data stored on your device is encrypted
  • We use AES-256 encryption for local data storage
  • Encryption keys are securely stored in your device's secure storage when available

2.2 Local Storage

Data stored locally on your device is protected:

  • Sensitive data is never stored in plain text
  • We minimize the amount of sensitive data stored locally
  • Temporary files are securely deleted when no longer needed
  • App data is isolated from other apps on your device

2.3 Data Synchronization

When synchronizing data with our servers:

  • All data is encrypted during transmission
  • We use secure API endpoints with proper authentication
  • Synchronization occurs only when necessary to minimize data exposure

3. Authentication & Access

3.1 User Authentication

We implement secure authentication methods:

  • Strong password requirements
  • Support for biometric authentication (fingerprint, face recognition) when available
  • Multi-factor authentication options
  • Secure token-based authentication
  • Automatic session timeout for inactivity

3.2 Authorization

We ensure proper authorization for all actions:

  • Role-based access controls
  • Principle of least privilege
  • Server-side validation of all requests

4. Device Security

4.1 Device Compatibility

Our app is designed to work securely on supported devices:

  • Compatible with modern Android and iOS versions
  • Optimized for devices with security features
  • Leverages platform security capabilities when available

4.2 App Permissions

We request only the permissions necessary for app functionality:

  • Internet access: To connect to our services
  • Storage: To save calculations and reports
  • Camera (optional): For document scanning
  • Notifications: For alerts and updates

You can manage these permissions through your device settings.

4.3 Screen Security

We implement measures to protect sensitive information on screen:

  • Option to hide sensitive data when app is in background
  • Prevention of screenshots for sensitive screens (where supported by the platform)
  • Automatic locking after inactivity

5. Offline Security

5.1 Offline Data Protection

When using the app offline, your data remains protected:

  • Encrypted local storage for offline data
  • Secure caching mechanisms
  • Authentication state maintained securely

5.2 Synchronization Security

When your device reconnects to the internet:

  • Secure synchronization of offline changes
  • Conflict resolution with data integrity checks
  • Verification of authentication before sync

6. Security Updates

6.1 App Updates

We regularly update the app to address security issues:

  • Security patches are released promptly
  • Critical security updates are prioritized
  • Automatic update notifications

6.2 Vulnerability Management

Our approach to managing vulnerabilities includes:

  • Regular security assessments
  • Third-party penetration testing
  • Monitoring of security advisories
  • Prompt remediation of identified issues

We encourage users to keep their app updated to the latest version to benefit from security improvements.

7. Security Best Practices

To enhance the security of your data when using our mobile app, we recommend the following practices:

7.1 Device Security

  • Keep your device's operating system and apps updated
  • Use a strong device passcode or biometric authentication
  • Enable remote wipe capabilities for your device
  • Install a reputable mobile security app
  • Avoid using the app on jailbroken or rooted devices

7.2 Account Security

  • Use a strong, unique password for your FinergyCloud account
  • Enable multi-factor authentication when available
  • Sign out when using shared devices
  • Regularly review your account activity

7.3 Network Security

  • Avoid using public Wi-Fi for sensitive transactions
  • Use a VPN when connecting to public networks
  • Verify you're connected to legitimate networks

7.4 Data Management

  • Regularly export and backup important data
  • Delete data you no longer need from the app
  • Be mindful of what information you share through the app

8. Contact Security Team

Email: security@finergycloud.com

We appreciate responsible disclosure of security vulnerabilities and will work with you to address any legitimate concerns.
